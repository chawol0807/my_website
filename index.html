Google Place API 爬蟲應用

本專案透過 Google Places API 蒐集台北市酒吧與甜點店資料，分析其地理分布，並以互動式熱點地圖呈現商業集中區域。

Python Google Places API Folium HeatMap

程式設計與技術應用

使用 requests 套件向 Google Places API 發送 HTTP 請求， 取得指定關鍵字（酒吧、甜點店）在特定範圍內的店家資料。 透過參數設定 location、radius 與 keyword， 控制資料搜尋的地理範圍與類型。
Google Places API 單次請求最多回傳 20 筆資料， 本專案透過 next_page_token 搭配迴圈， 分頁取得最多三頁資料，確保資料蒐集的完整性。
在爬蟲資料抓取時，加入 Try/Except 例外控制， 避免在執行過程中產生錯誤，導致程式無法運行。
將 API 回傳的資料轉換為 DataFrame， 並透過欄位拆解與資料清洗，建立適合地理視覺化的資料結構。
使用 Folium 建立互動式地圖，並結合 HeatMap 呈現商家密集程度。 同時透過不同顏色的標記區分酒吧與甜點店，提升視覺辨識度與互動性。

統整結論

Google資料查詢 -> AI工具分析 -> 個人觀察總結

一、整體市場現況

(依據熱點圖及佐證資料分析)
外部資料佐證:
  • 台北101購物中心 長期為全台百貨營收前段班，顯示區域消費力強。
  • 台北市房地市場動態 指出信義計畫區為台北市租金最高區段之一。
  • 品牌多集中於百貨體系與大型商場，市場成熟度高。
    • 推論結論:
  • 屬於「高消費 × 高競爭」的成熟市場。
  • 適合品牌型、資本型經營者，新創進入門檻較高。
    • 外部資料佐證:
  • 媒體報導 指出東區面臨高租金壓力與空店面問題。
  • 店面租金長期維持高檔水準。
  • 屬於夜生活成熟商圈，品牌與既有店家聚集。
    • 推論結論:
  • 夜生活成熟，但市場目前處於飽和走向衰退的過程。
  • 新進者需創造強烈差異化才能突圍。
    • 外部資料佐證:
  • 南京東路／中山商辦區 為台北主要商辦走廊之一。
  • 新聞報導 指出該區辦公室密集，餐飲多以商務客群為主。
  • 客群以平日上班族為主，假日人流較弱。
    • 推論結論:
  • 客源穩定但以商務型需求為主。
  • 屬於成熟穩定型市場，創新空間有限。
    • 外部資料佐證:
  • 台北捷運公司公開資料顯示⌈六張犁站⌋ 進站 出站 人流穩定，屬於生活型站點。
  • 根據 公開資料 顯示，該區域以居住與生活機能為主要發展方向，可推論具有較高的住宅人口比例。
  • 網路搜尋店面租金相較於以上區域明顯較低。
    • 推論結論:
  • 競爭相對低。
  • 租金負擔較小。
  • 社區型消費具穩定性。
  • 屬於潛力發展區。

    • 二、選址建議

    吳興街商圈、六張犁捷運站周邊、信義路五段靠住宅段
    1. 競爭密度低於核心商圈（熱點圖）
    2. 租金壓力較低（市場資訊）
    3. 捷運通勤人口穩定（捷運公開資料）
    4. 住宅混合型客群具下班小酌需求
    結論:適合社區型精品酒吧或特色小型品牌發展
    吳興街住宅區、象山捷運站周邊、六張犁文青區
    1. 住宅人口穩定(都市更新公開資訊)
    2. 象山站為生活型與觀光混合站點(捷運公開資料)
    3. 甜點業態需白天人流與女性客群支撐
    4. 生活圈型商圈較有日常消費基礎
    結論:適合日常型、社區型甜點品牌發展

    三、觀察總結

      1. 信義核心 = 高消費 × 高租金 × 高競爭
      2. 東區 = 成熟夜生活市場，進入存量競爭
      3. 南京商辦 = 客源穩定但成長有限
      4. 邊緣住宅混商區較有成長空間
      5. 核心外圍生活圈 = 成本低 × 成長潛力高
      根據空間熱點分布與公開租金、人流、市場資料綜合判斷： 若為新創品牌，建議選擇信義核心外圍生活圈，而非進入高度飽和的核心戰場。
